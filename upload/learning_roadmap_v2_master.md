## Sección 14: Seguridad para Automatizaciones e Inteligencia Artificial
> **Tagline:** *La automatización no protegida es una bomba de tiempo. La IA sin seguridad es una puerta trasera.*  
> **Nivel:** Competente | **Horas estimadas:** 10h | **Icono:** 🔒

### Relevancia laboral
La seguridad en automatizaciones e IA es el gap más grande del mercado peruano y LATAM. Las empresas US y globales requieren que sus ingenieros de AI/Automation entiendan el OWASP LLM Top 10, protejan pipelines de datos, y defiendan modelos contra ataques de prompt injection. En 2026, el 65% de las brechas de seguridad involucran algún componente de IA o automatización. Esta sección es obligatoria para cualquier perfil mid-senior.

### Objetivos de aprendizaje
- Entender y aplicar el OWASP LLM Top 10 (2025) a aplicaciones reales con IA
- Implementar defensa en capas contra prompt injection directa e indirecta
- Usar `presidio` de Microsoft para anonimización de PII en pipelines de datos
- Aplicar `detect-secrets` y gestión segura de credenciales con variables de entorno
- Entender hashing y criptografía con `hashlib`, `cryptography` (cubierto en Python 201 Ch. 14)
- Implementar logging de auditoría y trazabilidad en bots y agentes
- Aplicar el principio de mínimo privilegio en automatizaciones con APIs y bases de datos

### Temas de teoría
1. **OWASP LLM Top 10 (2025):** LLM01 Prompt Injection, LLM02 Sensitive Info Disclosure, LLM03 Supply Chain, LLM04 Data Poisoning, LLM05 Improper Output Handling, LLM06 Excessive Agency, LLM07 System Prompt Leakage, LLM08 Vector/Embedding Weaknesses, LLM09 Misinformation, LLM10 Unbounded Consumption
2. **Criptografía práctica en Python** (Python 201, Ch.14): `hashlib` para SHA-256 hashing, PBKDF2 para key derivation, `cryptography` con Fernet (symmetric) y RSA (asymmetric)
3. **Gestión de secretos:** `python-decouple`, `python-dotenv`, AWS Secrets Manager básico, nunca hardcodear API keys
4. **Anonimización y PII:** Microsoft `presidio` — detección y anonimización de NIT, DNI, correos, teléfonos peruanos en DataFrames
5. **Defensa contra prompt injection en 6 capas:** (1) input validation con regex, (2) system prompt restrictivo, (3) output filtering, (4) privilege control en tools, (5) human-in-the-loop para acciones críticas, (6) adversarial testing
6. **`detect-secrets` como pre-commit hook:** detectar API keys, passwords y tokens antes de hacer git commit
7. **Logging de auditoría con `structlog`:** logs inmutables, estructurados (JSON), trazables, con campos: timestamp, user_id, input_hash, output_hash, latencia, security_flags
8. **Seguridad en automatizaciones RPA:** sanitizar inputs web, rotar credenciales, usar storage_state de Playwright con cifrado

3. Un pipeline de verificación de integridad con `hashlib` SHA-256 para archivos de modelos ML
4. Configuración de `detect-secrets` como pre-commit hook en el repo del curso con `git commit` que falla si detecta un secret

## Sección 15: Python Avanzado — stdlib Profunda
> **Tagline:** *El Python que todos usan superficialmente. Tú lo dominarás en profundidad.*  
> **Nivel:** Competente | **Horas estimadas:** 12h | **Icono:** ⚙️

### Relevancia laboral
Las entrevistas técnicas de empresas US para roles Python Senior testean: context managers, functools, itertools, descriptors y type hints. Esta sección cubre íntegramente los capítulos de Python 201 que no tenían sección dedicada en la versión original del roadmap: Ch. 3 (contextlib), Ch. 4 (functools), Ch. 8 (itertools), Ch. 10 (typing), Ch. 12 (Unicode), Ch. 16-18 (super, descriptors, scope).

### Objetivos de aprendizaje
- Crear context managers con `contextlib`: `contextmanager`, `closing`, `suppress`, `ExitStack`
- Usar `functools` avanzado: `lru_cache`, `partial`, `singledispatch`, `wraps`, `reduce`
- Explotar `itertools`: infinite iterators (`count`, `cycle`, `repeat`), combinatorics (`product`, `combinations`, `permutations`), terminators (`groupby`, `islice`, `chain`)
- Implementar descriptors con `__get__`, `__set__`, `__delete__` para validación de atributos
- Aplicar type hints completos con `typing`: `Protocol`, `TypeVar`, `Generic`, `Literal`, `TypedDict`, `overload`
- Manejar Unicode y encoding correctamente en pipelines de datos con texto peruano (ñ, tildes, caracteres especiales)
- Comprender MRO y `super()` en herencia múltiple (diamond problem)
- Dominar scope: closures, `nonlocal`, fábricas de funciones

### Temas de teoría
1. **Context Managers** (Python 201, Ch. 3): `with` statement internals, `__enter__`/`__exit__`, `contextlib.contextmanager`, `contextlib.ExitStack` para múltiples CMs dinámicos
2. **functools avanzado** (Python 201, Ch. 4): `lru_cache(maxsize=128)` para memoización, `partial` para fijar argumentos (currying), `singledispatch` para function overloading por tipo, `wraps` para decorators bien formados que preservan `__name__` y docstring
3. **itertools arsenal** (Python 201, Ch. 8): `count(start, step)`, `cycle(iterable)`, `repeat(obj, times)`, `chain(*iterables)`, `islice(iterable, stop)`, `groupby(iterable, key)`, `product(*iterables, repeat)`, `combinations(iterable, r)`, `permutations(iterable, r)`
4. **Descriptors** (Python 201, Ch. 17): non-data vs data descriptors, `__set_name__`, uso en frameworks (Django fields, SQLAlchemy columns)
5. **Type Hints modernos** (Python 201, Ch. 10): `Protocol` para duck typing estructurado, `TypeVar` para genéricos, `Generic[T]`, `Literal["value"]`, `TypedDict`, `@overload` para múltiples firmas
6. **Unicode en profundidad** (Python 201, Ch. 12): code points, UTF-8 vs UTF-16, `str.encode()`, `bytes.decode()`, `chardet` para detectar encoding de archivos, problemas comunes con CSVs de sistemas legacy peruanos
7. **super() y MRO** (Python 201, Ch. 16): Algoritmo C3 linearization, `type.__mro__`, diamond problem resuelto
8. **Scope y closures** (Python 201, Ch. 18): LEGB rule, `nonlocal`, cell objects, factories de funciones para currying manual
